 # GrowthKit SDK 层级切换流程图

## 🎯 系统概述

GrowthKit SDK 实现了完整的四层级视图管理系统，通过智能的层级切换机制实现广告点击功能。整个系统完全黑盒化，外部无需了解内部实现细节。

## 📊 层级架构

```
┌─────────────────────────────────────────────────────────────┐
│                    四层级视图架构                              │
├─────────────────────────────────────────────────────────────┤
│ 层级3: 原生弹窗层 (zIndex: 200)                              │
│ ├── 引导弹窗 (CustomPopupView)                              │
│ └── 事件穿透按钮 (点击穿透到WebView)                          │
├─────────────────────────────────────────────────────────────┤
│ 层级2: 游戏视图层 (zIndex: 99)                               │
│ ├── Unity游戏内容 (外部提供)                                  │
│ └── 主要用户交互界面                                          │
├─────────────────────────────────────────────────────────────┤
│ 层级1: 单层WebView容器 (zIndex: 10)                          │
│ ├── 广告WebView (SingleLayerWebContainer)                   │
│ └── Unity截图遮盖层 (视觉连续性)                              │
├─────────────────────────────────────────────────────────────┤
│ 层级0: 多层WebView容器 (zIndex: 0)                           │
│ ├── WebView层1 (展示内容A)                                   │
│ ├── WebView层2 (展示内容B)                                   │
│ ├── WebView层3 (展示内容C)                                   │
│ └── 更多WebView... (依赖配置levelMax)                        │
└─────────────────────────────────────────────────────────────┘
```

## 🔄 层级切换流程

### 1. 正常游戏状态
```
┌─────────────────┐
│   3: 原生弹窗    │ ← 隐藏状态
├─────────────────┤
│   2: Unity游戏   │ ← 用户可见交互 (zIndex: 99)
├─────────────────┤  
│ 1: 广告WebView  │ ← 隐藏在Unity下方 (zIndex: 10)
├─────────────────┤
│ 0: 多层WebView  │ ← 隐藏运行 (zIndex: 0)
└─────────────────┘
```

### 2. 广告检测触发
```
┌─────────────────┐
│   3: 原生弹窗    │ ← 准备显示
├─────────────────┤
│   2: Unity游戏   │ ← 即将切换到下层
├─────────────────┤  
│ 1: 广告WebView  │ ← 即将切换到顶层
├─────────────────┤
│ 0: 多层WebView  │ ← 继续运行
└─────────────────┘
```

### 3. 层级切换完成
```
┌─────────────────┐
│   3: 原生弹窗    │ ← 显示引导 (zIndex: 200)
├─────────────────┤
│ 1: 广告WebView  │ ← 置顶显示 (zIndex: 99)
│   + Unity截屏   │   (截屏遮盖)
├─────────────────┤
│   2: Unity游戏   │ ← 切换到下层 (zIndex: 10)
├─────────────────┤
│ 0: 多层WebView  │ ← 继续运行 (zIndex: 0)
└─────────────────┘
```

### 4. 用户点击弹窗
```
┌─────────────────┐
│   3: 原生弹窗    │ ← 事件穿透到WebView
├─────────────────┤
│ 1: 广告WebView  │ ← 接收点击事件
│   + Unity截屏   │   (广告跳转)
├─────────────────┤
│   2: Unity游戏   │ ← 等待恢复
├─────────────────┤
│ 0: 多层WebView  │ ← 继续运行
└─────────────────┘
```

### 5. 层级恢复
```
┌─────────────────┐
│   3: 原生弹窗    │ ← 延迟关闭
├─────────────────┤
│   2: Unity游戏   │ ← 恢复到顶层 (zIndex: 99)
├─────────────────┤  
│ 1: 广告WebView  │ ← 切换回下层 (zIndex: 10)
├─────────────────┤
│ 0: 多层WebView  │ ← 继续运行 (zIndex: 0)
└─────────────────┘
```

## 🔧 技术实现要点

### 1. 事件穿透机制
- 弹窗按钮设置为可穿透
- 点击事件传递到下层WebView
- 确保广告点击的有效性

### 2. 截图遮盖技术
- 实时捕获Unity游戏画面
- 将截屏作为静态遮盖层
- 保持视觉连续性

### 3. 层级状态管理
- 使用@Published属性监控状态
- 防止重复切换
- 动画效果控制

### 4. 自动触发机制
- 监听广告检测状态
- 自动触发层级切换
- 无需外部干预

## 🎉 黑盒化特性

### ✅ 外部零感知
- 外部只需提供游戏视图和截图提供者
- 所有复杂逻辑内部处理
- 无需了解层级切换细节

### ✅ 自动管理
- 广告检测自动触发
- 层级切换自动执行
- 状态恢复自动处理

### ✅ 稳定可靠
- 防止频繁切换
- 状态同步机制
- 错误处理机制

# GrowthSDK 构建和发布使用说明

## 概述

GrowthSDK 采用**双仓库分离**的方式：
- **阿里云仓库**：只存放源代码（私有）
- **GitHub 仓库**：只存放发布文件（framework + podspec + README）

## 仓库结构

```
阿里云仓库 (codeup.aliyun.com)
├── GrowthSDK/          # 源代码
├── GrowthSDK.xcodeproj
├── GrowthSDK.xcworkspace
├── Podfile
├── scripts/            # 构建脚本
├── .gitignore          # 忽略构建产物
└── README.md

GitHub 仓库 (github.com)
├── Frameworks/
│   └── GrowthSDK.xcframework/
├── GrowthSDK.podspec
└── README.md
```

## 日常开发流程

### 1. 开发阶段

在 Sourcetree 中正常开发源代码：

```bash
# 修改源代码
# 在 Sourcetree 中提交
git add .
git commit -m "feat: add new feature"
git push origin main
```

**注意**：`Frameworks/` 目录已被添加到 `.gitignore`，不会提交到源代码仓库。

### 2. 清理构建产物（可选）

如果遇到构建问题，可以清理构建产物：

```bash
./scripts/clean-build.sh
```

### 3. 发布新版本

```bash
# 发布版本 1.0.0
./scripts/build-and-release.sh 1.0.0
```

## 使用方法

### 1. 准备工作

确保您已经：
- 在阿里云上有源代码仓库
- 在 GitHub 上创建了发布仓库：`shuge-developer/GrowthSDK`
- 更新了 `GrowthSDK.podspec` 中的 GitHub 仓库地址

### 2. 给脚本添加执行权限

```bash
chmod +x scripts/build-and-release.sh
chmod +x scripts/clean-build.sh
```

### 3. 发布新版本

```bash
# 发布版本 1.0.0
./scripts/build-and-release.sh 1.0.0

# 发布版本 1.1.0
./scripts/build-and-release.sh 1.1.0
```

### 4. 脚本执行流程

脚本会自动执行以下步骤：

1. **检查环境**：确保在阿里云仓库中
2. **添加远程**：自动添加 GitHub 远程仓库
3. **拉取代码**：从阿里云获取最新源代码
4. **创建目录**：确保 Frameworks 目录存在
5. **构建 framework**：使用 Xcode 构建 Release 版本
6. **创建 xcframework**：生成支持多架构的 framework
7. **更新版本**：修改 podspec 中的版本号
8. **提交源代码**：只提交源代码到阿里云仓库
9. **准备 GitHub**：克隆或更新 GitHub 仓库
10. **复制发布文件**：复制 framework、podspec 和 README
11. **提交到 GitHub**：推送到 GitHub 仓库
12. **创建标签**：创建版本标签

### 5. 下游集成

下游 APP 可以通过 CocoaPods 集成：

```ruby
# Podfile
pod 'GrowthSDK', '~> 1.0.0'
```

## 注意事项

### 1. .gitignore 配置

`.gitignore` 文件已配置忽略以下内容：
```
# Framework distribution (built artifacts)
Frameworks/
```

这意味着：
- ✅ 源代码会正常提交
- ❌ 构建产物不会提交到源代码仓库
- ✅ 发布时构建产物会推送到 GitHub 仓库

### 2. 远程仓库配置
- **origin**: 阿里云仓库（源代码）
- **github**: GitHub 仓库（发布文件）
- 脚本会自动添加 GitHub 远程仓库

### 3. 权限要求
- 需要阿里云仓库的推送权限
- 需要 GitHub 仓库的推送权限
- 确保 SSH 密钥配置正确

### 4. 构建要求
- 确保 Xcode 项目配置正确
- 确保所有依赖都已安装
- 确保有足够的磁盘空间

### 5. 文件分离
- **阿里云仓库**：不包含任何构建产物
- **GitHub 仓库**：只包含发布必需的文件

## 故障排除

### 远程仓库问题
```bash
# 查看远程仓库配置
git remote -v

# 手动添加 GitHub 远程仓库
git remote add github git@github.com:shuge-developer/GrowthSDK.git

# 测试推送权限
git push github main
```

### 构建失败
```bash
# 清理构建产物
./scripts/clean-build.sh

# 检查 Xcode 项目
open GrowthSDK.xcworkspace

# 手动构建测试
xcodebuild -project GrowthSDK.xcodeproj -scheme GrowthSDK -configuration Release build
```

### 权限问题
```bash
# 检查 SSH 密钥
ssh -T git@github.com
ssh -T git@codeup.aliyun.com
```

### 构建产物问题
```bash
# 如果看到 Frameworks 目录的变更
git status

# 确保 .gitignore 正确配置
cat .gitignore | grep Frameworks

# 如果已经提交了构建产物，需要移除
git rm -r --cached Frameworks/
git commit -m "remove build artifacts"
```

## 优势

1. **源代码保护**：源代码在阿里云私有仓库，不包含构建产物
2. **简化集成**：下游只需要通过 GitHub 获取 framework
3. **自动化**：一个脚本完成所有操作
4. **版本管理**：清晰的版本标签管理
5. **分离关注点**：开发环境和发布环境完全分离
6. **仓库清洁**：每个仓库只包含必要的文件
7. **日常开发友好**：构建产物不会干扰源代码提交

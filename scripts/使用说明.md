# GrowthSDK 构建和发布使用说明

## 概述

GrowthSDK 采用**双仓库分离**的方式：
- **阿里云仓库**：只存放源代码（私有）
- **GitHub 仓库**：只存放发布文件（framework + podspec + README）

## 仓库结构

```
阿里云仓库 (codeup.aliyun.com)
├── GrowthSDK/          # 源代码
├── GrowthSDK.xcodeproj
├── GrowthSDK.xcworkspace
├── Podfile
├── scripts/            # 构建脚本
├── .gitignore          # 忽略构建产物
└── README.md

GitHub 仓库 (github.com)
├── Frameworks/
│   └── GrowthSDK.xcframework/
├── GrowthSDK.podspec
└── README.md
```

## 日常开发流程

### 1. 开发阶段

在 Sourcetree 中正常开发源代码：

```bash
# 修改源代码
# 在 Sourcetree 中提交
git add .
git commit -m "feat: add new feature"
git push origin main
```

**注意**：`Frameworks/` 目录已被添加到 `.gitignore`，不会提交到源代码仓库。

### 2. 发布新版本

```bash
# 发布版本 1.0.0
./scripts/build-and-release.sh 1.0.0
```

### 3. 清理构建产物（仅在需要时）

**使用场景**：
- 遇到构建问题时排查
- 清理磁盘空间
- 重置构建环境

```bash
# 仅在需要时运行
./scripts/clean-build.sh
```

**注意**：日常开发通常不需要运行此脚本，因为 `Frameworks` 目录已被 `.gitignore` 忽略。

## 使用方法

### 1. 准备工作

确保您已经：
- 在阿里云上有源代码仓库
- 在 GitHub 上创建了发布仓库：`shuge-developer/GrowthSDK`
- 更新了 `GrowthSDK.podspec` 中的 GitHub 仓库地址

### 2. 给脚本添加执行权限

```bash
chmod +x scripts/build-and-release.sh
chmod +x scripts/build-sdk.sh
chmod +x scripts/clean-build.sh
```

### 3. 发布新版本

```bash
# 发布版本 1.0.0
./scripts/build-and-release.sh 1.0.0

# 发布版本 1.1.0
./scripts/build-and-release.sh 1.1.0
```

### 4. 单独构建 SDK（可选）

如果您只想构建 SDK 而不发布，可以使用：

```bash
# 基本构建
./scripts/build-sdk.sh

# 详细构建日志
./scripts/build-sdk.sh --verbose

# 调试版本构建
./scripts/build-sdk.sh -c Debug

# 自定义输出目录
./scripts/build-sdk.sh -o ~/Desktop/SDKs
```

### 5. 脚本执行流程

`build-and-release.sh` 会自动执行以下步骤：

1. **检查环境**：确保在阿里云仓库中
2. **添加远程**：自动添加 GitHub 远程仓库
3. **拉取代码**：从阿里云获取最新源代码
4. **专业构建**：调用 `build-sdk.sh` 进行专业 SDK 构建
5. **更新版本**：修改 podspec 中的版本号
6. **提交源代码**：只提交源代码到阿里云仓库
7. **准备 GitHub**：克隆或更新 GitHub 仓库
8. **复制发布文件**：复制 framework、podspec 和 README
9. **提交到 GitHub**：推送到 GitHub 仓库
10. **创建标签**：创建版本标签

### 6. 下游集成

下游 APP 可以通过 CocoaPods 集成：

```ruby
# Podfile
pod 'GrowthSDK', '~> 1.0.0'
```

## 注意事项

### 1. .gitignore 配置

`.gitignore` 文件已配置忽略以下内容：
```
# Framework distribution (built artifacts)
Frameworks/
```

这意味着：
- ✅ 源代码会正常提交
- ❌ 构建产物不会提交到源代码仓库
- ✅ 发布时构建产物会推送到 GitHub 仓库

### 2. 远程仓库配置
- **origin**: 阿里云仓库（源代码）
- **github**: GitHub 仓库（发布文件）
- 脚本会自动添加 GitHub 远程仓库

### 3. 权限要求
- 需要阿里云仓库的推送权限
- 需要 GitHub 仓库的推送权限
- 确保 SSH 密钥配置正确

### 4. 构建要求
- 确保 Xcode 项目配置正确
- 确保所有依赖都已安装
- 确保有足够的磁盘空间

### 5. 文件分离
- **阿里云仓库**：不包含任何构建产物
- **GitHub 仓库**：只包含发布必需的文件

## 故障排除

### 远程仓库问题
```bash
# 查看远程仓库配置
git remote -v

# 手动添加 GitHub 远程仓库
git remote add github git@github.com:shuge-developer/GrowthSDK.git

# 测试推送权限
git push github main
```

### 构建失败
```bash
# 清理构建产物
./scripts/clean-build.sh

# 检查 Xcode 项目
open GrowthSDK.xcworkspace

# 手动构建测试
xcodebuild -project GrowthSDK.xcodeproj -scheme GrowthSDK -configuration Release build
```

### 权限问题
```bash
# 检查 SSH 密钥
ssh -T git@github.com
ssh -T git@codeup.aliyun.com
```

### 构建产物问题
```bash
# 如果看到 Frameworks 目录的变更
git status

# 确保 .gitignore 正确配置
cat .gitignore | grep Frameworks

# 如果已经提交了构建产物，需要移除
git rm -r --cached Frameworks/
git commit -m "remove build artifacts"
```

## 文件管理

### 源代码仓库（阿里云）
包含以下文件：
- `GrowthSDK/` - SDK 源代码
- `GrowthSDK.xcodeproj/` - Xcode 项目文件
- `GrowthSDK.xcworkspace/` - CocoaPods 工作区
- `Podfile` - CocoaPods 依赖配置
- `Podfile.lock` - 依赖锁定文件
- `scripts/` - 构建和发布脚本
- `README.md` - SDK 文档（**新增**）
- `.gitignore` - Git 忽略配置

### 发布仓库（GitHub）
包含以下文件：
- `Frameworks/GrowthSDK.xcframework/` - 编译后的框架
- `GrowthSDK.podspec` - CocoaPods 配置文件
- `README.md` - SDK 文档（从源代码仓库复制）

### README 文件管理

**重要更新**：README 文件现在统一管理在源代码仓库中，发布时会自动复制到 GitHub 仓库。

**优势**：
- ✅ 便于维护和更新
- ✅ 版本控制
- ✅ 团队协作
- ✅ 自动同步到发布仓库

**更新流程**：
1. 在源代码仓库中编辑 `README.md`
2. 提交到阿里云仓库
3. 发布时自动复制到 GitHub 仓库
